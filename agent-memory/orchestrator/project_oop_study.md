오브젝트(조영호 저) 를 기반으로 OOP 개념을 스팸브레이커 코드와 연결하며 학습 중.

스터디 파일 위치: `docs/study/oop/`

**진행 현황 (2026-05-22 기준):**
- 01: 객체와 책임 (티켓 판매 시스템, 정보 전문가 패턴)
- 02: 메시지가 먼저다 (도서관 유즈케이스, 상태 vs 계산 트레이드오프)
- 03: 역할과 다형성 (역할/구현, 다형성, OCP, SRP)
- 04: SRP 실전 (클래스/메서드 단위 SRP, 추상화 레벨)
- 05: 의존성 관리하기 (결합도, DI, DIP, 명시적 의존성, 생성자 주입)
- 06: 상속 vs 합성 (화이트박스/블랙박스, 전략 패턴, 데코레이터 패턴, IS-A, LSP)
- 07: LSP 심화 + ISP (타입 계층 재설계, 사전/사후조건, 인터페이스 분리 기준)
- 08: 디자인 패턴 실전 (옵저버, 팩토리, 프록시, 데코레이터 vs 프록시, 패턴 조합)

**다음 후보:** 실전 리팩토링 (스팸브레이커 코드에 배운 패턴 직접 적용) 또는 추가 패턴 (커맨드, 템플릿 메서드 심화)

**Why:** 실무 코드(스팸브레이커) 품질 향상을 위한 OOP 원칙 내재화.
**How to apply:** 세션 시작 시 README.md와 마지막 파일 읽어서 어디까지 했는지 파악.
