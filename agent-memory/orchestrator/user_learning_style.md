설명이 완벽하지 않으면 "근데 왜?", "그거는 합성도 마찬가지잖아", "그러면 if가 계속 반복되지 않아?" 같이 끝까지 밀어붙인다.

"아 그렇구나" 하고 넘어가지 않는다. 논리적으로 납득이 안 되면 반드시 되묻는다.

억지로 끼워 맞춘 예시는 바로 잡아낸다. 설명이 불공평하거나 비교가 성립하지 않으면 지적한다.

실전 연결도 잘 한다 — "스프링 빈에서 전략 패턴 되긴 해?" 같이 배운 걸 바로 현실에 적용해서 의문을 던진다.

**How to apply:** 설명을 억지로 끼워 맞추지 말 것. 예시가 논리적으로 완벽하지 않으면 먼저 인정하고 수정할 것. 유저가 밀어붙일 때 방어하지 말고 같이 파고들 것.
