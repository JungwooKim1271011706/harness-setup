# 사람 검증 스크립트 (옆집 아저씨 기준) — 출력 템플릿

> 이 파일은 **커밋되는 출력 포맷 템플릿**이다. 리뷰모드(2차)가 검증 B(사람 스크립트)를 출력할 때 이 구조를 채워서 사용자에게 보여준다.
> 민감값(계정/PW/운영 토큰)은 이 파일에도, 채워진 출력에도 **평문으로 적지 않는다**. `.claude/review/scenarios.local.md` 참조 지시문 또는 `<…입력>` 플레이스홀더만 둔다(보안요건 c). 채워진 출력은 finalizer가 `gstack-redact`로 비밀 스캔해 평문 누출을 자동 치환한다(가드 — 절대 airtight 아님, 실수 차단용).
> `⚠ 봇MR 포맷 가정`: URL·라우트·클릭 대상의 자동 추출은 봇 MR 코드/로컬 기동 산출에 의존한다. 실샘플 확보 전까지 추출 규칙은 가정이며, 실샘플로 보정한다.

---

## 검증 대상

- **WI 번호**: WI-<번호>
- **WI 기대 행위 (oracle)**: <WI 본문의 "기대 행위" 항목을 그대로 인용 — 봇 코드가 아니라 이것이 정답>
- **대상 MR**: !<MR번호1>, !<MR번호2> … (WI에 걸린 전 MR)
- **기동 정보**: <접속 URL — 로컬 기동에서 자동 추출. 추출 실패 시 `scenarios.local.md` 참조>

---

## 사전 준비 (옆집 아저씨도 그대로 따라 할 수 있게)

1. 브라우저를 연다.
2. 주소창에 다음을 입력한다: `<기동 URL — 자동 추출>`
3. 로그인 화면이 나오면:
   - 아이디: `scenarios.local.md`의 `test_account_id` 값 입력 (이 문서에 평문 금지)
   - 비밀번호: `scenarios.local.md`의 `test_account_pw` 값 입력 (이 문서에 평문 금지)
   - `scenarios.local.md`가 없으면: `<테스트ID 입력>` / `<테스트PW 입력>` 로 사용자에게 요청

---

## 단계별 확인 (각 단계: 무엇을 클릭 → 무엇이 보여야 함)

> 아래 각 행은 봇 MR 코드(라우트·버튼 id·페이지 경로)와 로컬 기동에서 **자동 추출**해 채운다(값 출처 다).
> `⚠ 봇MR 포맷 가정`: 추출 규칙(예: `@RequestMapping` → 화면 경로, JSP onClick → 클릭 대상)은 이 프로젝트 컨벤션 가정에 의존한다.

| # | 무엇을 한다 (클릭/입력 대상) | 기대 결과 (눈으로 보이는 것) | WI oracle 근거 | 통과? |
|---|------------------------------|-------------------------------|----------------|-------|
| 1 | <메뉴/화면 경로 — 자동 추출>로 이동 | <화면이 떠야 함> | <WI 기대행위 N번> | ☐ |
| 2 | <클릭 대상 버튼/링크 — 자동 추출> 클릭 | <기대 변화 — WI 기준> | <WI 기대행위 N번> | ☐ |
| 3 | <입력 폼 — 자동 추출>에 <샘플 데이터(scenarios.local.md 참조)> 입력 후 저장 | <성공 메시지/목록 반영> | <WI 기대행위 N번> | ☐ |

> 샘플 데이터는 `scenarios.local.md`의 `sample_data` 항목에서 가져온다(평문 전개 금지, 참조만).

| # | 기존 흐름 (회귀 범위) | 기대: 변화 없음 | 통과? |
|---|------------------------|------------------|-------|
| 1 | <회귀 흐름 — WI 회귀범위 항목> | 기존과 동일하게 동작 | ☐ |

## 판정

- 모든 ☐가 통과 → 사람 스크립트 **PASS**
- 하나라도 실패 → 사람 스크립트 **FAIL** (→ 전체 FAIL, 아래 MR 댓글 템플릿 사용)
